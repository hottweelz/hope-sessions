# Asset And Proof Checklist

## Purpose

This checklist covers the materials needed across grants, Hope initiative outreach, and the future PMA supplemental page.

## Identity And Bio Assets

- approved short bio for Paul Andrejack
- approved long bio for Paul Andrejack
- concise PMA studio description
- Andrejack band description
- short description of the album Hope and why it matters to the initiative

## Work Sample Assets

- final list of strongest Andrejack tracks
- links to stream or embed work samples
- lyrics or excerpts if useful for storytelling
- release dates and roles for each sample
- any live or studio session footage worth reusing

## Proof And Credibility Assets

- selected past artist credits
- selected industry names or collaborations that can be publicly named
- testimonials that are safe to quote publicly
- media appearances or press mentions
- studio photos and portraits of Paul Andrejack

## Grant Assets

- residency proof
- resume or CV
- project description
- project timeline
- base budget inputs
- letters-of-support target list
- entity or tax documentation if required by the portal

## Hope Initiative Assets

- final working headline
- one-paragraph initiative description
- partner-facing explanation
- artist-facing explanation
- first-cycle project summary
- second-stage future-program summary

## Web Assets

- hero headline
- subheadline
- CTA labels
- section copy blocks
- one primary visual direction
- favicon, logo, or wordmark decision if needed
- inquiry form fields and routing plan
- approved PMA logo or wordmark asset
- approved Hope album cover asset

## Missing Or Unverified Items

- final flagship Hope project format
- approved work sample set for grants
- approved work sample set for public web use
- public-use image library
- permission status for quotes and testimonials
- confirmation of which credits can be named on the site
- explicit public-web reuse permission status for the Two Sides To Every Story child image

## Operating Rule

Any proof element used in public-facing materials should also be safe to cite in grant or partner contexts. Avoid building separate truth systems for the website and the application package.
